§ 02.02 · Systemic Crisis · ▲ PAIN

多体协作的系统性危机。

当 Agent 数量从 2 提升到 n，通信、决策、归因同时被放大——现有框架缺乏对这三类失败的系统级诊断工具。
▲ FAILURE 01

通信冗余

Agent 间反复复述已有上下文，token 成本线性甚至平方级增长；有效信息密度反而下降。

▲ FAILURE 02

协作冲突

当各 Agent 看到的上下文子集不同，会得出互相矛盾的判断；级联幻觉进一步放大错误。

▲ FAILURE 03

责任推诿

任务失败后难以定位到具体失败跳——每个 Agent 都在"合理地"响应上游输入。

§ 03.01 · Long-Context Decomposition

长上下文的任务分解 — 链式处理。

§ A · Chain of Agents

把长上下文拆成可接力的短上下文。

每个 Agent 只读一段原文，产出摘要或中间状态；下一跳接力处理。

  • 优势简单、可复现、不改模型结构
  • 劣势链越长信息衰减越严重
  • 适用长文档摘要、跨文档问答

§ 04.03 · Memory ⟷ Reasoning · ▲ PAIN

记忆与推理的断层。

当记忆系统做得更深之后，新的矛盾暴露出来：存储 和 推理 不是同一套数据结构。
▲ GAP 01 · Catastrophic Forgetting

灾难性遗忘

新记忆覆盖旧权重表征——这是神经网络的结构性问题。增量微调、LoRA 叠加、外挂向量库都只是缓解，不是根治。

▲ GAP 02 · RAG ↛ Reasoning

检索难以融入推理逻辑

RAG 返回的是被 贴进 prompt 的独立文本——不是被 织入 推理链的结构化事实。检索准确率高 ≠ 最终答案质量高。

§ 05.01 · Synthesis

三大维度的关联总结 — 三条边是同一个三角。

维度 代表工作 它怎样影响另外两条边
架构 AutoGen / MetaGPT SOP 用结构化产物绕开文本带宽瓶颈；但固定拓扑收紧了记忆演化的空间。
上下文 Chain of Agents · C2C · BudgetThinker 带宽决定了架构能组多大；反过来，算力预算又逼迫记忆做压缩。
存储 CoALA · A-MEM · Memory-R1 记忆的结构反过来塑造"可协作拓扑"——A-MEM 的链接图即一种长期架构。
三句话总结
架构决定上下文怎么流转，
上下文的带宽反过来约束架构能组多复杂，
而存储是上下文的长期版本。
三者互为边界条件。任一维度的突破，都会重新释放另外两维的设计空间。

§ 05.02 · Opportunity I

破局点一 — Agentic Workflow 的系统级鲁棒性诊断与容错。

问题 · 多 Agent 系统出错时，只能靠人肉 debug。

现有框架几乎没有标准化的失败归因、回滚与熔断机制；分布式系统里成熟的 tracing / circuit-breaker / retry 思路还没有被认真移植到 Agentic Workflow。

借鉴对象 · 分布式系统成熟经验
L1
每次 Agent 调用打 span，构建因果图 (causal graph)
L2
Confidence-aware fallback——信心过低自动触发备选路径
L3
可回滚的状态快照：局部失败不污染全局上下文
L4
端到端的系统级 benchmark：不仅测任务成功率，也测失败定位能力
偏工程、缺口明确、门槛可控 · 适合作为阶段性产出。
Figure · 占位 ［此处放置：Agentic Tracing 面板示意
每个 Agent 节点 → span 树 → 失败点高亮；
类比 OpenTelemetry / Jaeger UI］
可交付物 · 开源 Tracing SDK + 可视化 Dashboard。

§ 05.03 · Opportunity II

破局点二 — 物理层降维打击：异构模型间的高效 KV-Cache 语义通讯。

假设 · C2C 在同构模型间成立；异构模型间是下一座山。

Cache-to-Cache 已经证明：同构或近似同构的 LLM 之间，KV-Cache 可直接投影融合。但 Coder × Math × Vision 这种 异构 组合——隐层维度、注意力结构、词表都不同——如何对齐仍是开放问题。
